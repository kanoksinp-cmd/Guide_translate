# Smart Mail Desk ✉️

เว็บแอปเดียว (HTML/CSS/JS ล้วน ๆ) ที่แปลงข้อความตามสบายให้เป็น **อีเมลที่เป็นทางการ**
พร้อม **หน้าต่างแกะคำศัพท์ทีละคำในรูปประโยค** — ทำงานทั้งหมดในเบราว์เซอร์ของผู้ใช้
**ไม่ใช้ AI หรือ API ภายนอกใด ๆ** จึงเปิดใช้งานได้ทันทีหลังอัปโหลดขึ้น GitHub Pages
โดยไม่ต้องตั้งค่า backend หรือ API key

## คุณสมบัติ

- **สองโหมดการแปลง** เลือกได้ตามต้องการ:
  - **แปล + จัดรูปแบบ** — แปลข้อความข้ามภาษาแบบคำต่อคำ แล้วจัดเป็นอีเมล รองรับ 3 คู่ภาษา: **ไทย ↔ อังกฤษ**, **ญี่ปุ่น ↔ อังกฤษ**, **จีน ↔ อังกฤษ** (เลือกภาษาที่ไม่ใช่อังกฤษ แล้วเลือกทิศทางได้)
  - **จัดรูปแบบอย่างเดียว** — ไม่แปลภาษา แต่ปรับคำพูดกันเอง/คำลำลองให้เป็นคำที่เหมาะกับอีเมลธุรกิจ ในภาษาเดิม รองรับ **ไทยและอังกฤษเท่านั้น** (ตรวจภาษาจากตัวอักษรในข้อความให้อัตโนมัติ)
- **หน้าต่างแกะคำศัพท์** — เปิดดูความหมาย/คำแปลของแต่ละคำในประโยคต้นฉบับแบบ interlinear gloss (โหมดแปล) หรือดูรายการคำที่ถูกปรับให้เป็นทางการแบบ track-changes (โหมดจัดรูปแบบอย่างเดียว)
- **เทมเพลตอีเมลตามวัตถุประสงค์** — ทั่วไป, ขอความช่วยเหลือ, ติดตามงาน, ขอบคุณ, ขอโทษ/แจ้งปัญหา (มีคำลงท้ายและหัวข้อแนะนำให้อัตโนมัติ)
- ระบุผู้รับ/ผู้ส่ง/หัวข้อได้ และคัดลอกอีเมลที่ได้ด้วยปุ่มเดียว
- ไม่มีข้อมูลใดถูกส่งออกจากเบราว์เซอร์ของผู้ใช้เลย

## ทำงานอย่างไร (สำคัญ — โปรดอ่าน)

ระบบนี้ **ไม่ใช่ AI จริง** ตามที่ตั้งใจออกแบบไว้ แต่เป็น **พจนานุกรม + กฎ (rule-based) ที่เขียนด้วย JavaScript ล้วน**:

1. **ตัดคำ**: ภาษาไทย/ญี่ปุ่น/จีนไม่มีเว้นวรรคระหว่างคำ ระบบจึงตัดคำด้วยอัลกอริทึม greedy longest-match เทียบกับพจนานุกรมของภาษานั้น ๆ (ภาษาอังกฤษเว้นวรรคอยู่แล้วจึงไม่ต้องตัดคำเพิ่ม)
2. **แปล/ปรับคำ** โดยใช้ภาษาอังกฤษเป็นภาษากลาง (pivot) — แต่ละภาษาอื่นมีพจนานุกรมคู่กับอังกฤษของตัวเอง (ไทย ~250 คำ, ญี่ปุ่น/จีน ~100 คำ เน้นคำที่ใช้บ่อยในอีเมลธุรกิจ รวมถึงคำไวยากรณ์พื้นฐานเพื่อให้แปลคำผสมที่ไม่เคยเจอมาก่อนได้ดีขึ้น) — คำที่เป็นภาษาเดียวกับภาษาปลายทางอยู่แล้ว (เช่น พิมพ์ชื่อคนหรือคำภาษาอังกฤษแทรกในประโยคไทยตอนแปลเป็นอังกฤษ) จะถูกข้ามไปโดยไม่ถือว่าเป็นจุดที่พจนานุกรมไม่ครอบคลุม ส่วนทิศทางอังกฤษ → ภาษาอื่น ระบบจะลองจับคู่คำต่อเนื่อง 3, 2, แล้ว 1 คำก่อนแปล เพื่อให้วลีอย่าง "thank you" หรือ "purchase order" แปลเป็นคำเดียวในภาษาปลายทางได้ถูกต้อง ไม่ถูกแปลทีละคำแยกกัน
3. **ประกอบเป็นอีเมล** ด้วยเทมเพลตทักทาย/คำลงท้ายตามภาษาปลายทางและวัตถุประสงค์ที่เลือก (รูปแบบคำทักทาย/คำลงท้ายอ้างอิงตามธรรมเนียมทั่วไปของแต่ละภาษา เช่น "[ชื่อ] 様" ในญี่ปุ่น หรือ "此致/敬礼" ในจีน)

หมายเหตุสำหรับญี่ปุ่นและจีน: พจนานุกรมทั้งสองภาษานี้เพิ่มเข้ามาใหม่และเล็กกว่าไทยมาก โดยเฉพาะภาษาญี่ปุ่นที่มีการผันคำกริยาซับซ้อน (เช่น 送ります/送りました/送って ล้วนมาจากคำเดียวกัน) ระบบนี้เก็บรูปคำที่ผันแล้วเป็นคำอิสระแต่ละรูปในพจนานุกรม **ไม่ได้แยกวิเคราะห์ไวยากรณ์ภาษาญี่ปุ่นจริง ๆ** จึงตัดคำได้ไม่สมบูรณ์เท่าภาษาไทยสำหรับประโยคที่ใช้รูปกริยานอกเหนือจากที่มีอยู่ในพจนานุกรม

ข้อดี: ทำงานได้ทันทีแบบ static site ไม่ต้องมี API key หรือเซิร์ฟเวอร์
ข้อจำกัด: เป็นการแปลแบบคำต่อคำ **ไวยากรณ์และการเรียงคำอาจไม่เป็นธรรมชาติเท่าการแปลโดยคนหรือ AI จริง**
และคำที่ไม่อยู่ในพจนานุกรมจะถูกปล่อยผ่านโดยไม่แปล (หน้าต่างแกะคำศัพท์จะระบุไว้ว่า "ยังไม่มีคำนี้ในพจนานุกรม")

### อยากเปลี่ยนไปใช้ AI จริงในอนาคต?

ถ้าต้องการความแม่นยำที่สูงขึ้น สามารถแทนที่ฟังก์ชัน `translateText()` และ `toneUpgrade()`
ใน `script.js` ด้วยการเรียก API แปลภาษา/LLM จริงได้ (เช่น Anthropic API) แต่จะต้องมี backend
หรือ proxy เก็บ API key อย่างปลอดภัย เพราะ GitHub Pages เป็น static hosting ที่ไม่ควรฝัง API key
ไว้ในโค้ดฝั่ง client ตรง ๆ

## โครงสร้างไฟล์

```
.
├── index.html      หน้าเว็บหลัก (โครงสร้าง UI ทั้งหมด)
├── style.css       ดีไซน์ทั้งหมด (ธีม "โต๊ะเรียบเรียงจดหมาย")
├── script.js       พจนานุกรม + ตรรกะแปล/ปรับคำ + การเชื่อมต่อ UI ทั้งหมด
├── README.md       ไฟล์นี้
├── .gitignore      ไฟล์ที่ไม่ต้องการให้ Git ติดตาม
└── LICENSE         สัญญาอนุญาต MIT
```

ทั้งสามไฟล์หลัก (`index.html`, `style.css`, `script.js`) คือไฟล์เดียวที่จำเป็นสำหรับเว็บนี้
ไม่มี dependency หรือ build step ใด ๆ — เปิด `index.html` ตรง ๆ ก็ใช้งานได้

## วิธีรันบนเครื่องตัวเอง

ดับเบิลคลิกเปิด `index.html` ในเบราว์เซอร์ได้ทันที หรือถ้าต้องการรันผ่าน local server (แนะนำ เพื่อให้ฟอนต์/ทุกอย่างโหลดสมบูรณ์):

```bash
# Python 3
python3 -m http.server 8000
# แล้วเปิด http://localhost:8000
```

## วิธี deploy ขึ้น GitHub Pages

1. สร้าง repository ใหม่บน GitHub แล้ว push ไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้นไป (branch `main`)
2. ไปที่ **Settings → Pages**
3. ที่ **Build and deployment** เลือก source เป็น **Deploy from a branch**
4. เลือก branch `main` และโฟลเดอร์ `/ (root)` แล้วกด **Save**
5. รอประมาณ 1–2 นาที จะได้ลิงก์เว็บไซต์รูปแบบ `https://<username>.github.io/<repo-name>/`

เนื่องจากไม่มี backend หรือ API ภายนอก เว็บนี้จะทำงานได้ทันทีบน GitHub Pages โดยไม่ต้องตั้งค่าเพิ่มเติม

## ข้อจำกัดที่ทราบอยู่แล้ว

- คำศัพท์ในพจนานุกรมมีจำกัด — ไทย ~250 คำ, ญี่ปุ่นและจีน ~100 คำต่อภาษา (ใหม่กว่าและยังเล็กกว่าไทย) — ชื่อเฉพาะและตัวย่อ/รหัสที่ไม่เป็นมาตรฐาน (เช่นรหัสภายในบริษัท) จะไม่ถูกแปลโดยธรรมชาติ เพราะไม่มีทางอยู่ในพจนานุกรมใดได้ครบทุกคำ
- การแปลเป็นแบบคำต่อคำ ไม่มีการจัดเรียงไวยากรณ์ใหม่ จึงอาจได้ประโยคที่ฟังดูไม่เป็นธรรมชาติ โดยเฉพาะการแปลข้ามภาษาที่มีโครงสร้างไวยากรณ์ต่างกันมาก
- การตัดคำภาษาไทย/ญี่ปุ่น/จีนใช้วิธี longest-match แบบง่าย ซึ่งอาจตัดผิดได้กับคำที่ไม่อยู่ในพจนานุกรมหรือคำที่มีความกำกวม — ภาษาญี่ปุ่นมีการผันคำกริยาที่ระบบนี้จับคู่ได้เฉพาะรูปที่เก็บไว้ในพจนานุกรมเท่านั้น (ดูหัวข้อ "ทำงานอย่างไร" ด้านบน) จึงตัดคำได้หยาบกว่าไทยและจีนพอสมควร
- **โหมดจัดรูปแบบอย่างเดียว (ไม่แปลภาษา) รองรับเฉพาะไทยและอังกฤษ** ยังไม่มีสำหรับญี่ปุ่น/จีน เพราะการปรับระดับความเป็นทางการของทั้งสองภาษานี้ (เช่น keigo ในญี่ปุ่น) ต้องใช้กฎไวยากรณ์ที่ซับซ้อนกว่าพจนานุกรมคำต่อคำมาก
